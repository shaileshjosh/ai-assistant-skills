# Flutter PR Review

## When to use

Use when:

- Reviewing pull requests
- Reviewing branch changes
- Pre-merge validation

## Checklist

Verify:

- Architecture standards
- API standards
- Authentication standards
- Testing standards
- Security standards
- Null safety
- Error handling
- Riverpod usage
- Clean Architecture compliance

Provide:

1. Critical issues
2. Improvements
3. Security concerns
4. Test coverage gaps
5. Final approval recommendation
